Repository Architecture

The codebase is split around ownership boundaries. The client owns playback policy. The server owns HTTP and transfer state. Shared helpers stay in common modules.

Client Playback Pipeline

The player opens a file path or URL, discovers streams, starts audio and video workers, then keeps SDL rendering on the main thread.

OpenFFmpeg opens a local path or HTTP URL. If no path is passed, Windows file picker is used.
ProbeBest audio and video streams are selected. Codec contexts are created.
DemuxA worker reads packets and forwards audio packets to the audio worker.
DecodeVideo frames queue for rendering. Audio frames queue to SDL output.
RenderThe main thread uploads YUV or RGBA pixels to SDL textures and presents them.

Command Line

.\build\windows-msvc-vcpkg\Debug\webvideoplayback.exe http://127.0.0.1:8080/real-genius.mp4

Performance CSV writing is disabled by default.

.\build\windows-msvc-vcpkg\Debug\webvideoplayback.exe --performance-report http://127.0.0.1:8080/real-genius.mp4

Runtime Controls

  • F1 toggles the in-window debug overlay.
  • Esc exits playback.
  • WEBVIDEOPLAYBACK_AUDIO_LATENCY_MS adjusts audio target latency.
  • The default audio target is 150 milliseconds.

Test Server Pipeline

The server is a controlled local origin. It is not a full media platform. It exists to make client behavior observable and reproducible.

ConfigJSON maps public routes to exact local files.
ListenWinsock binds to loopback on the configured port.
AcceptEach client connection is handled on a worker thread.
ResolveRequests must match configured routes when a file list exists.
StreamFull or partial file bytes are sent with correct headers.

Streaming Contract

The player and server communicate using simple HTTP semantics. FFmpeg requests ranges as needed. The server responds with bytes from the configured file.

Initial read The client requests the configured route. The server serves the file or a byte range. It includes Accept-Ranges: bytes.
Range request The client can send Range: bytes=start-end. Valid ranges return 206 Partial Content.
Invalid range Out-of-bounds ranges return 416 Range Not Satisfiable with Content-Range: bytes */size.
Route policy When JSON file routes are configured, unlisted paths are denied. This keeps tests deterministic.

Timing And Synchronization

Audio is the master clock. Video waits for audio when it is early and drops stale frames only after a large delay.

Normal Playback

  • Audio decode feeds SDL queued audio.
  • A smoothed audio clock advances between SDL queue updates.
  • Video frame timestamps are compared against that clock.
  • Rendered frame timing can be written to CSV when enabled.

Window Interaction

  • Move and resize interactions pause playback.
  • Audio output pauses.
  • Demux and decode workers wait at a pause gate.
  • Playback resumes after mouse release or quiet resize events.

Support Guide

Use the dashboard, overlay, and optional CSV report together. The server explains transfer behavior. The player report explains decode, upload, render, and sync behavior.

Jerky Video

  • Enable --performance-report.
  • Check media_time_s for timestamp jumps.
  • Check present_ms for rendering stalls.
  • Check av_sync_ms for clock drift.

Server Issues

  • Confirm the route exists in JSON.
  • Confirm the file path exists locally.
  • Watch active stream progress.
  • Watch failed stream count.

Audio Problems

  • Look for increasing underrun counts.
  • Raise WEBVIDEOPLAYBACK_AUDIO_LATENCY_MS.
  • Inspect whether window interaction pause is active.

Window Interaction Problems

  • Move and resize intentionally pause playback.
  • Resume should happen after mouse release.
  • Resize has a quiet-period fallback.
